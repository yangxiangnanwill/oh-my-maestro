# Product

## Register

product

## Users

**目标用户**: 不习惯 CLI 的开发者，使用 Maestro 工作流编排但偏好可视化界面。

**使用场景**: 在 IDE-like 环境中管理项目工作流——查看项目状态、触发工作流、监控执行进度、与 AI 对话、使用嵌入终端。

**核心任务**: 快速了解项目状态 → 一键触发工作流 → 实时追踪步骤进度 → 审批关键操作 → 在终端/AI 对话中深入操作。

## Product Purpose

Maestro IDE 是基于 maestro-flow 的本地 Web 应用，将 CLI 工作流编排工具封装为可视化界面。核心理念：**概念翻译** — 将技术术语（chain/skill/delegate/session）映射为用户友好的概念（Workflow/Capability/AI Agent/Activity），让非 CLI 用户也能高效使用。

**成功标准**: 用户打开应用后能立即理解项目状态，一键触发工作流，无需学习 CLI 命令。

## Brand Personality

**精准、高效、可靠** — 专业工具感。

- **精准**: 信息展示精确，状态同步实时，不给用户模糊或过时信息
- **高效**: 操作路径最短，不阻塞用户流程，审批/确认内联展示
- **可靠**: 不崩溃、不丢数据、不静默失败，错误信息清晰可操作

参考工具: Linear（简洁精准的项目管理）、Figma（专业但不复杂的工具感）、Raycast（高效快捷的命令面板）

## Anti-references

- ❌ **Hacker 风格**: 避免绿字黑底终端美学、Matrix 风格、过度技术化的视觉语言
- ❌ **企业软件臃肿感**: 避免复杂菜单栏、多层工具栏、密密麻麻的表格和表单
- ❌ **游戏化 UI**: 避免过度动画、霓虹色、gamification 元素

## Design Principles

1. **自解释 (Self-Explanatory)**: 用户无需阅读文档即可理解每个控件的用途。图标 + 标签 + tooltip 三重保障。概念翻译层确保无术语泄漏。

2. **非阻塞 (Non-Blocking)**: 不阻塞用户操作流。审批/确认内联展示（非 modal），终端和 AI 对话可并行操作，状态更新实时推送。

3. **信息透明 (Information Transparency)**: 信息密度适中——不隐藏关键信息（如工作流步骤详情），也不过度展示（简单模式隐藏技术细节）。用户始终知道系统在做什么。

4. **一致性 (Consistency)**: 所有面板使用统一的事件/状态模式、相同的 Catppuccin Mocha 主题、一致的交互模式（展开/折叠、确认/取消、状态指示器）。

## Accessibility & Inclusion

- **WCAG 2.1 AA**: 颜色对比度 ≥ 4.5:1（普通文本）、≥ 3:1（大文本）
- **键盘可操作**: 所有交互元素可通过键盘访问和操作
- **屏幕阅读器友好**: 使用语义化 HTML、ARIA 标签、合理的 focus 管理
- **减少动画**: 尊重 `prefers-reduced-motion`，关键动画有静态替代
- **颜色不是唯一信息载体**: 状态指示器配合图标/文本，不仅依赖颜色
