### Implementation Steps
1. Create src/lib/server/dialog-manager.ts. Import { spawn, ChildProcess } from node:child_process. Import { EventBus } from ./event-bus.js. Import { DialogSession, WorkflowMeta, StreamChunk, IntentResult } from ../shared/types.js. Import { Channels, DialogEvents } from ../shared/events.js. Define SpawnFn type: (command: string, args?: readonly string[], options?: Parameters<typeof spawn>[2]) => ChildProcess (compatible with node:child_process.spawn full signature).
2. Define local types. StreamChunk: { type: text | tool_use | tool_result | error; content: string; timestamp: string }. IntentResult: { workflowId: string | null; confidence: number; candidates: Array<{ workflowId: string; score: number }> }. IntentCandidate: { workflowId: string; score: number }. ActiveDialog: { process: ChildProcess | null; session: DialogSession; buffer: string }.
3. Implement DialogManager class. Constructor signature: constructor(private eventBus: EventBus, private spawnFn: SpawnFn = spawn, private workflowRegistry: WorkflowMeta[] = []). Private members: private sessions = new Map(), private sessionCounter = 0, private readonly MAX_SESSIONS = 5.
4. Implement createSession(clientId: string): DialogSession. Check this.sessions.size >= this.MAX_SESSIONS, throw Error if so. Generate sessionId = dialog-{++this.sessionCounter}-{Date.now()}. Create DialogSession: { sessionId, cliProcessId: null, createdAt: new Date().toISOString(), lastActivityAt: new Date().toISOString(), status: active }. Store to sessions. Emit SESSION_CREATED: this.eventBus.publish(DialogEvents.SESSION_CREATED, Channels.DIALOG, { sessionId, clientId, session }, server). Return session.
5. Implement sendMessage(sessionId: string, message: string): void. Get activeDialog, throw Error if not found. Update lastActivityAt. If process is null, call this.spawnClaudeCLI(sessionId) to start. Write message + newline to activeDialog.process.stdin. Call this.detectIntent(message) for intent recognition.
6. Implement spawnClaudeCLI(sessionId: string): ChildProcess. const childProcess = this.spawnFn(claude, [--output-format, stream-json, --verbose]). Store to sessions process field. Set stdout data handler: split by lines, call parseChunk to parse, emitStreamChunk to emit. Set stderr handler: log error. Set exit handler: update session.status to closed, emit SESSION_CLOSED: this.eventBus.publish(DialogEvents.SESSION_CLOSED, Channels.DIALOG, { sessionId, exitCode: code }, server). Return childProcess.
7. Implement parseChunk(line: string): StreamChunk | null. Trim then return null if empty. try JSON.parse. Validate type field in [text, tool_use, tool_result, error]. Return { type: parsed.type, content: parsed.content ?? parsed.text ?? , timestamp: parsed.timestamp ?? new Date().toISOString() }. catch return null.
8. Implement emitStreamChunk(sessionId: string, chunk: StreamChunk): void. this.eventBus.publish(DialogEvents.STREAM_CHUNK, Channels.DIALOG, { sessionId, ...chunk }, server).
9. Implement detectIntent(message: string): IntentResult. If workflowRegistry is empty return { workflowId: null, confidence: 0, candidates: [] }. For each workflow compute keyword match score: (1) check if message contains workflow.name (case-insensitive) — +0.5; (2) check if contains keywords from workflow.description — each match +0.1; (3) check if contains workflow.category — +0.2. Normalize scores to 0-1. Sort candidates. If top score >= 0.8: emit INTENT_ROUTED (workflowId = candidates[0].workflowId, confidence). If top score 0.5-0.8: emit INTENT_ROUTED (workflowId = null, confidence = top score, candidates = top 3). Return { workflowId: bestMatch?.workflowId ?? null, confidence: bestMatch?.score ?? 0, candidates: top 3 }.
10. Implement closeSession(sessionId: string): void. Get activeDialog, silently return if not found. If process exists then process.kill(). Delete from sessions. Emit SESSION_CLOSED.
11. Create src/lib/server/__tests__/dialog-manager.test.ts. Use vi.mock(node:child_process) mock spawn. Construct mock workflowRegistry: [{ id: test-build, name: build, category: deploy, description: build the project, params: [] }, { id: test-lint, name: lint, category: quality, description: run linter, params: [] }]. Tests: (1) createSession returns DialogSession and emits SESSION_CREATED; (2) createSession 6th call throws Error; (3) sendMessage first call starts CLI subprocess; (4) stdout NDJSON line emits STREAM_CHUNK; (5) detectIntent build this project computes confidence >= 0.8 for test-build and emits INTENT_ROUTED; (6) detectIntent do something confidence 0.5-0.8 emits INTENT_ROUTED with candidates; (7) detectIntent xyzzy confidence < 0.5 only streams output no INTENT_ROUTED; (8) closeSession kills subprocess and emits SESSION_CLOSED.

### Convergence Criteria
- src/lib/server/dialog-manager.ts contains export class DialogManager
- DialogManager constructor accepts (eventBus: EventBus, spawnFn?: SpawnFn, workflowRegistry?: WorkflowMeta[]) params
- createSession(clientId) returns DialogSession object (with sessionId, cliProcessId: null, createdAt, status: active)
- createSession throws Error(Maximum dialog sessions (5) reached) when exceeding maxSessions=5
- sendMessage(sessionId, message) starts CLI via spawnClaudeCLI if not started, then writes message to process.stdin
- spawnClaudeCLI calls spawnFn(claude, [--output-format, stream-json, --verbose])
- stdout line parsing: parseChunk parses NDJSON lines, extracts type/content/timestamp fields, invalid lines return null
- STREAM_CHUNK event emission: eventBus.publish(DialogEvents.STREAM_CHUNK, Channels.DIALOG, { sessionId, type, content, timestamp }, server)
- detectIntent(message) returns IntentResult object: { workflowId: string | null; confidence: number; candidates: { workflowId, score }[] }
- confidence >= 0.8 emits DialogEvents.INTENT_ROUTED: { sessionId, workflowId, confidence }
- confidence 0.5-0.8 emits same event with candidates disambiguation list: { sessionId, workflowId: null, confidence, candidates }
- closeSession(sessionId) kills subprocess, deletes session, emits DialogEvents.SESSION_CLOSED
- updateWorkflowRegistry(workflows: WorkflowMeta[]) updates internal workflowRegistry array
- src/lib/server/__tests__/dialog-manager.test.ts exists and all tests pass
